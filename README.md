## 🧠 What I've Learned So Far

- **Control flow is everything.** Most bugs aren't typos — they're misunderstandings of how code *flows*.
- **Boundary conditions matter.** "Greater than" vs "greater than or equal" is the difference between working code and silent bugs.
- **DRY (Don't Repeat Yourself).** When you copy-paste a line with a tiny tweak — that tweak wants to be a variable.
- **Trace before you run.** Mentally walk through your code with a sample input. Find bugs on paper, not at runtime.
